## 新的操作流程

### 第1步：生成模板（自动包含区域编码）
```bash
python create_excel_template.py
```
✓ Excel文件中已包含有效的区域编码列表

### 第2步：填写数据
1. 打开 `e:/safety/data/batch_update_template.xlsx`
2. 查看"使用说明"页，确认有效的区域编码
3. **确保填写的区域编码来自该列表**
4. 填写消防器材和微型消防站数据
5. 保存文件

### 第3步：验证数据（新步骤！）
```bash
python validate_excel_data.py
```

这将检查：
- 区域编码是否有效
- 必填字段是否完整
- 日期格式是否正确

**必须验证通过才能继续导入！**

### 第4步：执行导入
```bash
python batch_update_database.py
```

导入时会：
1. 自动加载有效的区域编码
2. 对每一条数据进行验证
3. 无效数据会被拒绝
4. 详细记录导入结果

### 第5步：验证结果
- 查看导入日志
- 登入系统验证数据
- 确认所有数据已正确导入

## 常见问题

### Q1: 区域编码从哪里来？
**A:** 区域编码来自微型消防站物资负责人表（ResponsiblePerson）。
- 目前系统中有13个有效的区域编码（1-13）
- 所有新数据的区域编码必须使用这些值
- 可通过运行 `python get_valid_area_codes.py` 查看完整列表

### Q2: 如果导入失败，提示区域编码无效，怎么办？
**A:** 
1. 检查Excel中的区域编码是否在有效列表中
2. 确保没有多余的空格或特殊字符
3. 修正错误的区域编码
4. 重新运行 `python validate_excel_data.py` 验证
5. 验证通过后再导入

### Q3: 可以向系统中添加新的区域编码吗？
**A:** 理论上可以，但需要在ResponsiblePerson表中添加负责人记录。
- 请通过系统的管理界面添加新的负责人
- 一旦添加，新的区域编码会自动出现在导入模板中
- 新生成的Excel模板会包含最新的区域编码

### Q4: 验证脚本会影响数据库吗？
**A:** 不会。`validate_excel_data.py` 只读不写，完全安全。
- 它只是检查数据的有效性
- 不会修改或删除任何数据库记录
- 可以多次运行
